## Help System Sync

If the project has an in-app help system (contextual help, empty states, guided tours): **a user-facing feature is incomplete until its help reflects it** — missing or stale help is a regression, fixed in the same commit.

**Before declaring a user-facing change complete**, run the checklist in `docs/process/help-sync.md`; the project's help architecture, content schemas, and tone guidelines live in `docs/help.md`.
